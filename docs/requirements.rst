要件定義書
============

プロジェクト概要
----------------
Ames Housing データセットを用いて、住宅価格の要因分析と価格予測を行うデスクトップアプリケーションを開発する。

機能要件
--------

1. **データセットの読み込みと管理**
   - OpenMLから `house_prices` データセットを取得する。
   - 取得したデータは初回起動時にローカルに `house_prices.csv` としてキャッシュし、次回以降はキャッシュを使用する。

2. **データ閲覧機能 (Data)**
   - 取得したデータセットを表形式 (`DataTable`) で表示する。
   - 項目名および一部のデータ値（カテゴリ値）を日本語に翻訳して表示する。
   - パフォーマンスを考慮し、初期表示は先頭50件とする。

3. **分析機能 (Analysis)**
   - 住宅価格 (`SalePrice`) との相関が高い項目トップ10を分析し、棒グラフで可視化する。
   - 各項目の相関係数を表示する。
   - 専門知識がないユーザー向けに、分析手法の解説を表示する。

4. **価格予測機能 (Prediction)**
   - ユーザーが入力した条件に基づいて、住宅価格を予測する。
   - 使用するモデル: ランダムフォレスト回帰 (`RandomForestRegressor`)。
   - 入力項目:

     - 居住面積 (GrLivArea)
     - 建築年 (YearBuilt)
     - 地下室面積 (TotalBsmtSF)
     - 全体的な品質 (OverallQual)
     - ガレージ収容台数 (GarageCars)
   - 予測結果を表示するとともに、モデルがどの特徴量を重視したか（Feature Importance）を簡易グラフで表示する。

5. **欠損値分析機能 (Missing Values)**
   - データセット内の欠損値（空欄）を集計する。
   - 欠損値が多い項目トップ15を棒グラフで可視化する。
   - リスト形式でも欠損数と割合を表示する。

非機能要件
----------
- **UI/UX**: PythonのUIフレームワーク `Flet` を使用し、モダンでインタラクティブなGUIを提供する。
- **言語**: アプリケーション内のラベルや説明文は原則として日本語とする。
- **パフォーマンス**: データの読み込みや計算がUIをフリーズさせないよう配慮する（現状は同期処理だが、要件としてはスムーズな動作を目指す）。
